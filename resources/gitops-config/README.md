# GitOps Configuration - ArgoCD Bootstrap

This directory contains OpenTofu/Terraform configuration for bootstrapping the GitOps stack on a Talos Kubernetes cluster.

## Overview

The GitOps bootstrap solves the **chicken-and-egg problem** of deploying ArgoCD:
1. ArgoCD needs GitHub credentials to sync private repositories
2. GitHub credentials should be managed by External Secrets Operator (ESO)
3. ESO needs 1Password Connect to fetch secrets
4. But ArgoCD should manage both 1Password and ESO via GitOps

**Solution**: Terraform bootstraps the minimal stack, then ArgoCD takes over management.

## Bootstrap Flow

### Phase 1: Terraform Bootstrap (One-time)
1. ✅ Create namespaces (onepassword, external-secrets, argocd, longhorn-system)
2. ✅ Deploy 1Password Connect credentials secret (from input-files/)
3. ✅ Install 1Password Connect (Helm)
4. ✅ Install External Secrets Operator (Helm)
5. ✅ Create ClusterSecretStore (connects ESO to 1Password)
6. ✅ Deploy ExternalSecrets for GitHub credentials (from input-files/)
7. ✅ Install ArgoCD (Helm) - can now access GitHub
8. ✅ Install ArgoCD Apps/Projects (Helm)
9. ✅ Deploy sync-app (root ArgoCD Application)

### Phase 2: ArgoCD Takes Over (GitOps)
- sync-app deploys ArgoCD Applications that manage:
  - Longhorn (sync-wave 0)
  - Cilium (sync-wave 0)
  - 1Password Connect (sync-wave 1) - **replaces Terraform deployment**
  - External Secrets (sync-wave 2) - **replaces Terraform deployment**
  - Cert-manager (sync-wave 3)
  - ArgoCD itself (self-management)

## Architecture

```
resources/gitops-config/
├── *.tf                      # Terraform configuration
├── input-files/              # Sensitive secret files (gitignored)
│   ├── secret.yaml           # 1Password Connect credentials
│   ├── github-client-secret.yaml
│   ├── github-private-repo-creds.yaml
│   └── onepassword-connect-credentials.yaml
├── operators/                # Operator Helm values & templates
│   ├── argo-cd/
│   ├── external-secrets/
│   ├── longhorn/
│   ├── cert-manager/
│   └── cilium/
└── sync-app/                 # Root ArgoCD Application (App of Apps)
    └── templates/            # ArgoCD Application manifests
        ├── longhorn.yaml
        ├── cilium.yaml
        ├── onepassword-connect.yaml
        ├── external-secrets.yaml
        ├── cert-manager.yaml
        ├── argocd.yaml
        └── argocd-apps.yaml
```

## Prerequisites

### Required Tools
- [OpenTofu](https://opentofu.org/) or Terraform >= 1.0
- [kubectl](https://kubernetes.io/docs/tasks/tools/)
- [1Password CLI](https://developer.1password.com/docs/cli/get-started/)

### Required Access
- Kubernetes cluster with kubeconfig (from bootstrap step)
- 1Password Connect credentials in **KubernetesSecrets** vault
- GitHub App credentials in **KubernetesSecrets** vault
- Backblaze B2 credentials in **Homelab** vault (for S3 backend)

### Cluster Requirements
- Talos Kubernetes cluster must be running
- Cilium CNI must be installed (via bootstrap)
- Gateway API CRDs must be present

## Quick Start

### 1. Generate Input Files

Automatically generate all input-files from 1Password:

```bash
cd resources/gitops-config

# Ensure you're signed in to 1Password CLI
eval $(op signin)

# Generate all input-files
./scripts/generate-input-files.sh
```

**Required secrets in 1Password "KubernetesSecrets" vault:**
- `onepassword-connect-credentials.json` - 1Password Connect credentials (base64 encoded)
- `onepassword-connect-token` - 1Password Connect token
- `github-client-secrets` - GitHub OAuth client secret (for ArgoCD SSO)
- `github-argo-app` - GitHub App private key (for private repository access)

The script will automatically:
- ✅ Create `input-files/secret.yaml` with 1Password Connect credentials
- ✅ Create `input-files/github-client-secret.yaml` ExternalSecret
- ✅ Create `input-files/github-private-repo-creds.yaml` ExternalSecret
- ✅ Create `input-files/onepassword-connect-credentials.yaml` ExternalSecret

### 2. Set Environment Variables

Load Backblaze B2 credentials for S3 backend:

```bash
# Using 1Password CLI
cd resources/gitops-config
source ./scripts/load-secrets.sh

```

**Or use the bootstrap script** (if you already sourced it for bootstrap, no need to run again):
```bash
cd ../bootstrap
source ./scripts/load-secrets.sh
```

### 3. Configure Kubeconfig Path

Point to your cluster's kubeconfig (generated by bootstrap):

```bash
# Export path to kubeconfig
export TF_VAR_kube_config_path="$(pwd)/../bootstrap/output/kube-config.yaml"

# Or create terraform.tfvars
echo 'kube_config_path = "../bootstrap/output/kube-config.yaml"' > terraform.tfvars
```

### 4. Deploy GitOps Stack

```bash
# Initialize Terraform with S3 backend
tofu init

# Review the plan
tofu plan

# Deploy the bootstrap
tofu apply

# Verify ArgoCD is running
kubectl -n argocd get pods
kubectl -n argocd get applications
```

## Configuration

### Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `kube_config_path` | Path to kubeconfig file | `~/.kube/config` |
| `secret_path` | Path to 1Password credentials secret | `./input-files/secret.yaml` |
| `onepassword_version` | 1Password Connect Helm chart version | `1.17.0` |
| `external_secrets_version` | External Secrets Operator version | `0.17.0` |
| `argocd_version` | ArgoCD Helm chart version | `9.2.2` |
| `argocd_apps_version` | ArgoCD Apps Helm chart version | `2.0.2` |
| `namespaces` | Namespaces to create | `["onepassword", "external-secrets", "argocd"]` |

### Input Files

All files in `input-files/` must be created from `.example` templates:

#### secret.yaml
Contains the 1Password Connect credentials secret (base64 encoded JSON).

#### github-client-secret.yaml
ExternalSecret for GitHub OAuth client credentials (ArgoCD SSO).

#### github-private-repo-creds.yaml
ExternalSecret for GitHub PAT to access private repositories.

#### onepassword-connect-credentials.yaml
ExternalSecret to refresh 1Password credentials via ESO (after bootstrap).

## Accessing ArgoCD

### Get Initial Admin Password

```bash
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d
```

### Port Forward to ArgoCD UI

```bash
kubectl -n argocd port-forward svc/argo-cd-argocd-server 8080:443
```

Then open: https://localhost:8080

### Login via CLI

```bash
argocd login localhost:8080 --username admin --password <password>
```

## Sync Waves

ArgoCD applications are deployed in order using sync-waves:

- **Wave 0**: Longhorn, Cilium (infrastructure)
- **Wave 1**: 1Password Connect (secrets provider)
- **Wave 2**: External Secrets Operator (secrets sync)
- **Wave 3**: Cert-manager (certificates)

## Troubleshooting

### 1Password Connect Not Starting

Check if credentials secret exists:
```bash
kubectl -n onepassword get secret onepassword-connect-credentials
```

If missing, verify `input-files/secret.yaml` is correct.

### External Secrets Not Syncing

Check ClusterSecretStore status:
```bash
kubectl get clustersecretstore onepassword-connect -o yaml
```

Common issues:
- 1Password Connect not ready
- Wrong credentials in secret
- Network connectivity to 1Password service

### ArgoCD Can't Access GitHub

Check ExternalSecret status:
```bash
kubectl -n argocd get externalsecret github-private-repo-creds -o yaml
kubectl -n argocd get secret github-private-repo-creds
```

Verify:
- ClusterSecretStore is ready
- 1Password item references are correct
- GitHub PAT has correct permissions (repo read)

### Sync-App Not Deploying

Check if sync-app was created:
```bash
kubectl -n argocd get application sync-app
```

If missing, check Terraform null_resource:
```bash
tofu state list | grep deploy_root_app
```

### State File Issues

If `tofu init` fails with S3 errors:
- Verify AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY are set
- Check Backblaze B2 bucket exists: `homelab-prd`
- Confirm endpoint is correct: `s3.eu-central-003.backblazeb2.com`

## State Management

### S3 Backend

State is stored in Backblaze B2 (S3-compatible):
- Bucket: `homelab-prd`
- Key: `tofu/gitops-config.tfstate`
- Region: `eu-central-003`

**Credentials**: Uses same AWS_* environment variables as bootstrap.

### Migrating from Local State

If you have existing local state:

```bash
# Backup local state
cp terraform.tfstate terraform.tfstate.backup

# Configure backend in providers.tf (already done)

# Initialize and migrate
tofu init -migrate-state

# Verify state is in S3
tofu state list
```

## Managing the Stack

### Updating Versions

Edit `variable.tf` defaults or override in `terraform.tfvars`:

```hcl
# terraform.tfvars
onepassword_version     = "1.18.0"
external_secrets_version = "0.18.0"
argocd_version          = "9.3.0"
```

Then apply:
```bash
tofu plan
tofu apply
```

### Adding New ArgoCD Applications

1. Create application manifest in `sync-app/templates/`
2. Set appropriate sync-wave annotation
3. Reference operator values in `operators/<name>/`
4. Commit and push to GitHub
5. ArgoCD will auto-sync

### Destroying the Stack

**WARNING**: This will destroy ArgoCD and all managed applications!

```bash
# First, delete all ArgoCD applications
kubectl -n argocd delete applications --all

# Wait for resources to be cleaned up
kubectl get applications -A

# Then destroy Terraform resources
tofu destroy
```

## Security Considerations

### Secrets Management

- **Never commit** files in `input-files/` (except `.example` files)
- Use 1Password CLI to generate secrets programmatically
- Rotate GitHub PATs regularly
- Limit PAT scope to minimum required permissions

### Network Security

- ArgoCD UI should be behind authentication gateway
- Consider enabling ArgoCD SSO (GitHub OAuth configured)
- Use NetworkPolicies to restrict traffic
- Enable TLS for all ArgoCD connections

### RBAC

- Review ArgoCD project permissions
- Limit cluster-admin access
- Use namespace-scoped permissions where possible
- Enable ArgoCD RBAC policies
